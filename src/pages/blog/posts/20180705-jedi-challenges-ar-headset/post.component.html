Jedi Challenges

So yeah, it's a Star Wars Augmented Reality experience. While I am more interested in VR, I really wanted to try this out, but it was pretty expensive at the time. Luckily, I managed to get a used copy that was in a really good state.

I seriously wasn't expecting too much, but the whole thing is better than I expected. As you can see from the video, it's powered by the mobile. You put your phone in a container, and place it in the AR Headset. The result is that the display is mirrored onto the helmet itself, giving an Augmented Reality-like experience. The two cameras in the headset itself are able to track the beacon to determine your position. While it IS jittery, the fact that you can see the environment around you ensures that you don't get sick.

As for the games themselves: The Lightsaber battles are easily the best. You get indications on how to hold your weapon to block attacks and get cues on when to attack. While it starts off simply with Maul, the difficulty rate slowly goes up and you'll find yourself dodging and rotating the saber to block a flurry of attacks. I've only finished the first tier so far, and already having issues with the later stages. For what sorta looks like a novelty, cash-in title, it is surprisingly challenging.

The other modes are, overall, pretty well done as well, although they feel more like extras. Haven't done much with the chess mini-game, but the strategy game feels really cool with a holographic display of the battlefield. While it feels a bit odd to give commands with a lightsaber, it is a surprisingly good package.

I've also found it to be a good demo to bring to meetups. It's all mobile, easily fitting within a bag. And with the tracking beacon in there, it gives a good demonstration on how a Room Scale experience works. In fact, I would say that the AR doesn't add too much to the game. You never really feel as if Darth Vader is right there in the living room, and you quickly tend to forget about the environment as it is and focus more on the game itself.

Still, here's another confession: I didn't get this just for the game. First off, I wanted a Lightsaber prop in my office and I was told that the controller is a surprisingly good replica. Which, in my opinion, it is. But I accelerated my plans to get one when I saw the following video...

History repeating itself

I won't beat around the bush: I wouldn't be writing this article if I wasn't able to get something working. Still, it was a bit of a challenge. Here's how I managed to get it to work.

The AR Template

I started writing about the technical details on how I got the template to work, but I decided to finish writing all that for another post. It just got too technical, and frankly, I would like to start out with some background details.

You'll notice that the canvas object only takes up a small part of the scene. This is the only part that gets mirrored on the headset. It's also rendered as a stereo-effect for each eye. As mentioned in the above paragraph, everything that is black will be transparent. It works like a charm.

There are a couple of things other things I should mention...

  • You will need to run the page in full-screen mode. There is a manifest.json file which lets you create a bookmark on an Android device. After pressing it, the page will appear without the URL bar.
  • I'm using NoSleep.js to make sure that the screen doesn't go to sleep. After putting the device in the holder, you can't do any interaction with the mobile, so this is an easy way to keep it running.
  • The screen needs to be set to vertical mode. I tried to get this working via the manifest.json, but wasn't able to. Easiest way to do it is to put your device in landscape mode and then lock the orientation.

Using those few steps, you can build an AFrame project and get it running on a Lenovo headset. I will ollow up this post with a more technical one, where I will also post a link to a Github repository with a template.

What have I built with this?

Here's two projects that I've created with this.

Presentation notes

This is actually one of the things I had wanted to build as a prototype for the Vuzix when I worked at AR Development, but unfortunately, never took the opportunity.

I've given a few talks by now, and what I would like is for my presentation notes to be presented in front of me during the talk and synced to the slides. Using reveal.js and socket.io, I built a simple webapp that allows me to see those notes while wearing the AR Headset. I'm not even using AFrame here, but pure HTML to render the text.

Practical with a device like the Lenovo? Most likely not. But it's still an experience I want to try out at a meetup at some point and I have to admit that it actually felt pretty cool when training for a talk at home. I had hoped to use this application for a talk at role=drinks, but I unfortunately had to cancel at the last moment for personal reasons.

Still, if I can get the chance to give a talk at an informal meetup, I'll definitely see if I can use this.

AFrame scene previewer

So here's a fun little morning excercise.

I've ported my @aframevr live-demo viewer to the @lenovo headset that comes with Jedi Challenges. I can now edit a scene, and directly see the results by looking aside.

Honestly, this is the sort of thing that I wanna see in the future. pic.twitter.com/koqLqNK6Tq

This is a pretty good example of me wanting to use this headset for somethimg more practical and to get a preview on how I imagine 3D will be built and modelled in the future.

To accommodate the WebVR-related talks, I've developed a simple webapp that will update an A-Scene live by keeping the HTML in sync with a Firebase database. For this prototype, I've gone with a more advanced version that keeps an eye on the HTML files on my computer, and if they're updated, it will automatically update the a-scene without refreshing the page. Again, this was accomplished with node.js

What this means is that I can develop an AFrame project in my IDE, and while wearing the Lenovo headset, see the results happen in real-time to my right. I suppose this is one of those "Have to try it to believe it" things, but for me, this really makes building an AFrame scene so much easier. Instead of making a guess while looking at a screen or taking the time to wear/remove the headset, I can just instantly see the results, make a change in the HTML, save, and instantly see the results. I get a far better insight in what I'm building when seeing it being projected right in front of me, while still being able to code on my computer.

F.A.Q.

In conclusion, I want to share some final thoughts on this...

  • Any support for positional tracking or making use of the lightsaber? None.
  • Is it practical? No.
  • Could you see this being used seriously in a professional way? Nope.
  • Will it let you prototype AR ideas? Oh yeah!

And you know what? That last thing really makes the trouble worth it.
